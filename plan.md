# 사주 학습 모드 — 구현 계획 (plan.md)

> 작성일: 2026-06-10
> 근거: `research.md` (시장 공백 + 기존 자산 적합도 + 듀오링고형 학습 루프 + SM-2)

---

## 1. 목표와 방향

**기존 saju_app에 학습 모드(`/learn`)를 추가**한다. 별도 앱 분리 없이 계산 엔진·지식베이스·LLM 인프라를 공유한다.

핵심 컨셉: **"내 사주로 배우는 명리학"**
- 표준 커리큘럼(음양오행 → 천간지지 → 지장간 → 십성 → 12운성·신살 → 합충 → 실전)을 챕터로 제공
- 계산 엔진 기반 **퀴즈 무한 자동 생성** (정적 문제은행 불필요)
- 오답·암기 카드는 **SM-2 간격 반복**으로 복습
- 막히면 **AI 튜터**에게 질문 (기존 LLM 폴백 체인 재활용)

## 2. 아키텍처

```
backend-fastapi/
  learn_curriculum.py   # 커리큘럼 정의 (챕터·레슨·개념카드, data/ MD와 매핑)
  learn_quiz.py         # 퀴즈 생성기 (saju_utils 매핑 테이블 기반 동적 출제 + 채점용 해설)
  main.py               # /learn/* 라우트 추가
  ai_report.py          # 튜터 모드 프롬프트 함수 추가 (기존 LLM 호출 재사용)

frontend-next/src/
  app/(main)/learn/page.tsx              # 커리큘럼 맵 (챕터 목록 + 진도)
  app/(main)/learn/[chapterId]/page.tsx  # 챕터 학습 (개념 카드 → 퀴즈 → 결과)
  app/(main)/learn/review/page.tsx       # SM-2 복습 (오답·암기 카드)
  components/learn/*                     # 학습 전용 컴포넌트
  lib/learn.ts                           # localStorage 진도·SRS 관리 + API 호출
```

### API 설계
| 메서드 | 경로 | 기능 |
|---|---|---|
| GET | `/learn/curriculum` | 챕터 목록 (id, 제목, 설명, 개념 수, 권장 순서) |
| GET | `/learn/chapter/{chapter_id}` | 챕터 상세 — 개념 카드 목록 |
| GET | `/learn/quiz/{chapter_id}?count=N&seed=S` | 챕터별 퀴즈 N문항 동적 생성 (4지선다 + 정답 + 해설) |
| POST | `/learn/tutor` | AI 튜터 질의응답 (chapter 컨텍스트 + 지식베이스 주입, 스트리밍) |

### 진도·SRS 저장
- 기존 앱 관례(저장 명식 = localStorage)에 맞춰 **localStorage** 사용. 백엔드 DB 없음.
- `maek`이 아닌 saju_app 전용 키: `saju_learn_progress`(챕터별 상태·점수·XP·스트릭), `saju_learn_srs`(카드별 SM-2 상태)
- SM-2: 카드 = 퀴즈에서 틀린 문제 + 챕터 핵심 암기 카드. `{ef, interval, reps, due}` 보관, 프론트에서 계산.

각 챕터 = **개념 카드(학습) → 퀴즈(확인) → 통과 시 다음 챕터 잠금 해제**. 개념 카드 본문은 learn_curriculum.py에 요약 수록(레슨용 재가공), 깊은 내용은 AI 튜터가 지식베이스로 보충.

## 4. 퀴즈 생성기 설계 (learn_quiz.py)

문제 유형 (챕터별 출제 함수, `random.Random(seed)`로 재현 가능):
1. **오행 판별**: "丙의 오행은?" / "木을 생(生)하는 오행은?" (상생상극)
2. **천간·지지 속성**: 한자↔한글 읽기, 음양, 물상 키워드, 지지의 계절·시각·띠
3. **60갑자 순서**: "甲子 다음 간지는?", 특정 간지의 오행 조합
4. **지장간**: "寅의 지장간(정기)은?", 전체 구성 맞히기
5. **십성**: "일간이 甲일 때 庚은?" — GAN_TEN_GODS 그대로 (600문항 풀)
6. **12운성**: "甲 일간이 亥를 만나면?" — TWELVE_GROWTH (120문항 풀)
7. **합충형파해**: "子와 충하는 지지는?", "甲己는 무슨 관계?"
8. **실전**: 랜덤 생년월일 → calculate → "이 명식의 일간은?" "월지 십성은?"

오답 선지: 같은 카테고리 풀에서 정답 제외 랜덤 3개. 해설: 매핑 근거 + SAJU_TERMS 설명 결합.

## 5. AI 튜터 (ai_report.py 확장)

- `analysis_type="tutor"` 추가: 시스템 프롬프트를 "풀이 대가"에서 **"명리학 과외 선생님"**으로 교체
- 컨텍스트: 현재 챕터 관련 지식베이스 섹션 우선 주입 + 학습자가 틀린 문제 정보 전달 가능
- 출력: 짧은 문답형 (4섹션 헤딩 강제 없음), easy 난이도 어투
- 스트리밍 endpoint 재사용 (`/learn/tutor` → ai_report.stream 계열)

## 6. 프론트 학습 루프 (듀오링고 패턴 적용)

1. **커리큘럼 맵**: 챕터 카드 세로 나열, 완료 체크·진행률·잠금 표시 + 상단에 XP·스트릭(연속 학습일)
2. **챕터 학습**: 개념 카드 스와이프/넘기기 → "퀴즈 시작" → 1문항씩 즉각 정오 피드백+해설 → 결과 화면(점수, 틀린 문제는 복습 덱에 자동 추가) → 80% 이상 통과 시 다음 챕터 해제
3. **복습 탭**: due 카드만 노출, 알았다/헷갈림/몰랐다 3버튼 → SM-2 갱신
4. **AI 튜터**: 챕터 화면에서 "선생님께 질문" → 스트리밍 답변
5. (선택) 홈 화면의 내 명식과 연동: "내 사주로 보면?" 버튼

## 7. 구현 단계

- **Phase 1 (이번 작업, MVP)**: 백엔드 learn_curriculum/learn_quiz/라우트 + 프론트 커리큘럼 맵·챕터 학습·퀴즈·진도 저장 + 복습(SM-2) + AI 튜터
- **Phase 2 (후속)**: 실전 명식 읽기 고도화(내 명식 연동), 스트릭 프리즈·레벨 시스템, 학습 통계 화면
- **Phase 3 (후속)**: 신강신약·격국용신 고급 챕터, 통변 훈련(AI 채점), 공유 기능
